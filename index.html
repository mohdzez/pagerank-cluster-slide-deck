Section 1 — Introduction02 / 20
What is PageRank?
Speaker: Munim Ahmad
PageRank was developed by Larry Page and Sergey Brin at Stanford University to rank web pages in search results. It counts the number and quality of links to a page to estimate importance.
Core Assumption Important websites naturally receive more incoming links from other sites.
Iterative Process
01InitRank = 1/N per node
02DistributeSplit rank across outgoing edges
03DampenApply d = 0.85 factor
04ConvergeRepeat until stable
Formula
pagerank-formula
PR(A) = (1-d)/N + d * Σ( PR(Ti) / C(Ti) )

where:
  d     = damping factor (0.85)
  N     = total number of nodes
  Ti    = pages linking to A
  C(Ti) = outgoing links from Ti
Spark RDD Operations
rdd-operations
map       → transform each element
flatMap   → distribute rank contributions
reduceByKey → aggregate by node ID
join      → reconnect with link structure
cache()   → keep RDD in memory
Section 1 — Introduction03 / 20
Why Distributed Computing?
The Stanford Web-Google dataset contains millions of vertices and billions of edges. Single-machine processing would take hours. Apache Spark partitions the graph across multiple nodes for parallel execution.
Vertices 875K Web pages in the graph
Edges 5.1M Hyperlinks between pages
Format Adj. List Source ID → destination IDs
Distributed Advantages
  • Split the graph across machines (partitioning)
  • Process partitions in parallel across the cluster
  • Iterative algorithms converge faster through parallel execution
  • Fault tolerance via RDD lineage and checkpointing

WSL2 hides behind a private virtual network by default. Forcing Mirrored Mode strips this away, allowing the Linux subsystem to share the Windows laptop's physical IP address.

Section 4 — Implementation17 / 20
Performance Optimizations
Broadcast Variables
sc.broadcast() efficiently shares read-only data (graph structure) across all executors. Avoids re-shipping the adjacency list on every iteration.
Caching Strategy
.persist(StorageLevel.MEMORY_AND_DISK) handles datasets larger than available memory. Intermediate RDDs stay in memory; overflow spills to disk instead of recomputing.
Lazy Evaluation
Spark only computes when an action is triggered (collect, saveAsTextFile). Transformations are chained into an optimized execution plan.
reduceByKey vs groupByKey
reduceByKey performs local reduction before shuffling across the network. Significantly reduces network I/O compared to groupByKey which shuffles all data first.
Comparison
shuffle comparison
# groupByKey — shuffles ALL values
rdd.groupByKey().mapValues(sum)
# Network: O(n) data transferred

# reduceByKey — local reduce first
rdd.reduceByKey(lambda a, b: a + b)
# Network: O(k) data transferred
# where k = number of unique keys
